##Ciclo de la request https
Cuando accedés a una URL específica (como /mis-favoritos/), tu navegador envía esa petición al servidor.
El programa que está corriendo la web (Django, en tu caso) está configurado (en urls.py) para que, al acceder a esa URL en particular, se ejecute automáticamente una función de Python (tu vista en views.py, como getAllFavouritesByUser).
Cuando esa función de Python corre, su trabajo es obtener y preparar los datos dinámicos que se necesitan para esa página (por ejemplo, la id del Pokémon, la imagen, el nombre, altura, peso, etc., todo lo que tu función de servicio getAllFavourites devuelve).
Estos datos se organizan en un diccionario de contexto en Python y se le pasan a la función render().
Entonces, el motor de plantillas de Django toma el archivo HTML (favourites.html en este caso) y lo recorre. A medida que lo recorre, busca las variables y etiquetas especiales ({{ algo }} o {% for algo %}).
Mientras lo recorre, autocompleta esos "espacios en blanco" en el HTML con los datos que ya están cargados en el diccionario de contexto que le pasó la vista.
El resultado es un HTML final y completo que es enviado a tu navegador para que lo muestre.
